Practice Tailwind

SET UP

Liste des commandes à faire dans l'ordre :

  1. npm init -y
  2. npm install -D tailwindcss
  3. npx tailwindcss init
  4. faire les liens avec les pages, dans tailwind.config.js
  5. ajouter les directives dans src/styles.css
    • @tailwind base;
    • @tailwind components;
    • @tailwind utilities;
  6. lier les pages de styles entre src et public :
      7. npx tailwindcss -i ./src/styles.css -o ./public/styles.css --watch
  7. lier les pages html avec la page de style public
      8. link href="/public/styles.css" rel="stylesheet"

Just in Time TAILWINDCSS

Just in time add v2

pour ajouter JIT, et purger tous les styles qui ne sont pas utilisés par les fichiers html, on va ajouter les commandes --JIT --purge="./*html" lorsque l'on va lier les styles.css

npx tailwindcss -i ./src/styles.css -o ./public/styles.css --JIT --purge="./*html"

si on ajoute de nouvelles classes, il va falloir relancer la commande pour avoir les changements

en dvpt, rajouter le --watch à la fin de la commande, pour qu'elle s'execute en arrière plan

utiliser ["./*html"] dans le purge du tailwind.config pour appliquer le tailwind à tous les fichiers html

commande valable aussi pour la v3

Add CSS variables

pour ajouter des variables CSS réutilisables, on peut les créer dans le public/styles.css

on crée des variables dans le :root, et on les nomme avec deux traits devant : --

on peut ensuite appeler les variables n'importe où dans la feuille de style. On peut également nester les variables, en définit une première variable, suivi de var(seconde variable)

ex:

--bg-color-1: #1f2937

--bg-color-2: var(--bg-color-1)

on peut aussi créer ses propres classes dans le theme de tailwind.config puis les exporter dans le root

on peut aussi importer le thème par défaut de tailwind

tailwind default theme
